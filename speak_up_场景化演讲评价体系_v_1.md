# Speak Up 场景化演讲评价体系与量化指标库 V1.0

## 1. 文档目的

本文件用于为 **Speak Up** 建立一套统一、专业、场景化的演讲评价框架，作为产品设计、评分规则配置、实时反馈策略和赛后报告生成的基础。

本体系希望解决两个核心问题：

1. **如何在不同表达场景下进行更准确的评价**
2. **如何将专业评价标准转化为可落地的评分与反馈逻辑**

因此，Speak Up 的评价设计采用“**统一底层框架 + 场景化策略 + 量化指标支撑**”的思路：
- 用统一框架保证评价稳定、可比
- 用场景化策略保证评价贴合任务目标
- 用量化指标提升评分、反馈和成长追踪的可解释性

---

## 2. 设计原则

### 2.1 统一性原则
无论是主持、分享、即兴表达还是答辩，演讲评价都需要一个相对稳定的底层能力模型。只有底层框架统一，系统才能形成可持续的个人能力画像，并支持跨场景比较。

### 2.2 场景化原则
不同场景对表达者的核心要求不同。主持强调控场与互动，主题分享强调结构与解释，即兴表达强调快速组织与稳定输出，汇报答辩强调结论导向与专业回应，面试/Pitch 强调第一印象、亮点与说服力。因此，评价不能采用“一把尺子量到底”的方式，而应在统一底层框架上进行场景化权重调整。

### 2.3 专业性原则
本体系关注的不只是语言表面流畅度，而是综合考察内容组织、语言质量、表达节奏、声音表现、非语言呈现和听众影响力等多个维度，使其更接近真实演讲训练、正式陈述与公众表达的专业要求。

### 2.4 可解释性原则
评分结果必须能够解释。系统不仅要告诉用户“分数是多少”，更要告诉用户“为什么是这个分数”“优势在哪里”“最需要改进什么”。因此，评价体系应同时包含维度描述、场景重点和量化指标。

### 2.5 成长性原则
评价的目的不是一次性判断优劣，而是帮助用户持续提升。Speak Up 的评价结果应当服务于长期训练闭环，包括本次表现、相较上次的变化、近期趋势以及短板修复情况。

---

## 3. 评价框架

Speak Up 采用六个一级维度作为统一底层评价框架。

| 一级维度 | 核心含义 |
|---|---|
| 内容结构 | 是否有清晰主题、合理逻辑与完整收束 |
| 语言表达 | 用词、句式、准确性与简洁性是否得体 |
| 流利度与节奏 | 表达是否顺畅，语速与停顿是否合理 |
| 语音表现 | 发音、音量、语调、重音是否支持表达 |
| 非语言呈现 | 眼神、表情、姿态、手势是否自然有效 |
| 听众感与影响力 | 是否能吸引、带动、说服并影响听众 |

### 3.1 一级维度说明

#### 内容结构
关注表达是否围绕明确主题展开，是否具备开头、主体、结尾，是否能突出重点并完成总结收束。

#### 语言表达
关注语言使用是否准确、清楚、简洁、得体，是否存在大量空泛表达、重复表达或明显语言错误。

#### 流利度与节奏
关注表达过程是否稳定顺畅，语速是否合适，停顿是否自然，是否频繁出现卡顿、自我修正或节奏失衡。

#### 语音表现
关注发音是否清晰，音量是否稳定，语调是否具备必要起伏，是否能够通过声音强调关键信息。

#### 非语言呈现
关注眼神、表情、手势、姿态等非语言信号是否自然协调，是否支持整体表达效果。

#### 听众感与影响力
关注表达者是否具备听众意识，能否建立交流感，是否具有吸引力、说服力、感染力与记忆点。

## 4. 评分方式

本体系建议采用 **100 分制**，并通过 **5 档等级**解释评价结果。

| 等级 | 描述 |
|---|---|
| 5 卓越 | 表现稳定成熟，目标完成度高，短板很少 |
| 4 良好 | 主要要求达到，整体表现较成熟，存在少量优化空间 |
| 3 合格 | 基本完成表达任务，但在一个或多个方面仍有明显提升空间 |
| 2 待提升 | 存在较多问题，已影响表达效果或听众理解 |
| 1 基础不足 | 尚未完成该场景的核心表达任务 |

### 4.1 评分建议
- 每个场景仍以六个一级维度为评价基础。
- 不同场景通过权重变化体现任务重点。
- 具体评分可由“一级维度分 + 二级指标支撑 + 量化指标辅助”共同组成。

## 7. 场景评分表

## 7.1 主持（Host）

### 场景目标
主持场景的核心任务不是输出大量知识，而是建立场域、调动听众、衔接流程并保持节奏稳定。

### 评分表
| 一级维度 | 关键二级指标 | 分值 |
|---|---|---:|
| 听众感与影响力 | 开场吸引力、互动意识、现场带动感 | 25 |
| 流利度与节奏 | 节奏稳定性、转场流畅度、时间推进感 | 20 |
| 语音表现 | 音量控制、语调变化、声音感染力 | 20 |
| 非语言呈现 | 眼神交流、表情与姿态、手势配合 | 15 |
| 内容结构 | 主持流程完整性、主线清晰度 | 10 |
| 语言表达 | 用语得体性、语言简洁性 | 10 |

### 高分表现
控场自然，互动明显，转场顺滑，节奏与声音均能带动现场。

### 常见低分风险
开场慌乱、互动弱、转场生硬、节奏拖沓、感染力不足。

### 反馈重点
控场节奏、互动意识、转场自然度、声音带动感。

### 场景目标
主题分享的核心任务是把一个主题讲清楚，让听众能够理解、抓住重点并形成基本记忆。

### 评分表
| 一级维度 | 关键二级指标 | 分值 |
|---|---|---:|

| 语言表达 | 解释清晰度、用词准确性、表达简洁性 | 20 |
| 流利度与节奏 | 连贯性、停顿合理性、节奏层次 | 15 |
| 听众感与影响力 | 听众意识、吸引力、记忆点 | 15 |
| 语音表现 | 发音清晰度、语调控制、音量稳定性 | 10 |
| 非语言呈现 | 眼神交流、姿态与表情、手势辅助 | 10 |

### 高分表现
主题清楚，结构严密，解释充分，重点鲜明，结尾有力。

### 常见低分风险

### 反馈重点

## 7.3 即兴表达（Impromptu）

### 场景目标
即兴表达的核心任务是在有限准备时间下迅速起步、保持输出不断线，并形成基本清晰的表达主线。

### 评分表
| 一级维度 | 关键二级指标 | 分值 |
|---|---|---:|
| 流利度与节奏 | 起步反应速度、连续输出能力、停顿控制 | 30 |
| 内容结构 | 中心明确性、基本逻辑组织、结论收束 | 20 |
| 听众感与影响力 | 自然交流感、吸引力、稳定感 | 15 |
| 语音表现 | 发音与清晰度、音量稳定、语调自然度 | 15 |
| 语言表达 | 表达自然度、简洁与准确 | 10 |
| 非语言呈现 | 姿态稳定性、眼神与表情、整体呈现感 | 10 |

### 高分表现
起步快，表达稳，逻辑基本清楚，在有限准备下仍显自然。

### 常见低分风险
起步困难、频繁卡顿、组织跳跃、紧张失控、缺少结论。

### 反馈重点
起步速度、卡顿情况、即时组织能力、自然稳定度。

## 8. 实时反馈应用原则

完整评价体系主要用于 **赛后复盘**；实时反馈只应抽取其中少量、可即时纠偏的指标。

### 8.1 实时反馈的目标
实时反馈的核心不是实时给出完整评分，而是帮助用户在表达过程中及时发现明显偏差，并做出即时调整。

### 8.2 实时反馈优先指标

| 指标 | 推荐用途 | 典型提示方向 |
|---|---|---|
| 语速 | 全场景 | 过快/过慢提醒 |
| 长停顿次数 | 全场景 | 卡顿、停顿过长提醒 |
| 起步延迟时间 | 即兴、面试 | 尽快进入表达提醒 |
| 音量稳定性 | 全场景 | 声音偏低/不稳提醒 |
| 发音清晰度 | 全场景 | 发音不清提醒 |
| 注视镜头比例 | 全场景 | 注意镜头交流提醒 |
| 低头看稿比例 | 分享、答辩、面试 | 低头时间偏长提醒 |
| 口头禅密度 | 全场景 | 重复“嗯、啊、然后”等提醒 |
| 跑题偏移率 | 分享、答辩 | 提醒回到主题 |
| 单调度 | 主持、Pitch | 建议增加语调变化 |
| 互动触发率 | 主持、分享 | 建议加入听众呼应 |

### 8.3 实时反馈设计原则
- 一次只提示最关键的一项问题，避免频繁打断。
- 优先提示可立刻修正的问题，如语速、停顿、音量、镜头交流。
- 结构性、整体性问题更适合放在赛后报告中统一反馈。

## 9. 演讲评价量化指标库 V1.0

### 9.1 指标分类说明
量化指标分为两类：
- **核心评分指标**：可直接进入评分体系。
- **辅助诊断指标**：用于解释分数、生成建议、追踪成长，不建议在 V1.0 中高权重直接计分。

同时，每项指标可对应三种用途：
- 实时反馈
- 赛后复盘
- 长期趋势追踪

## 12. 关键量化指标定义表

| 指标名称 | 定义 | 建议计算方式 | 主要场景 |
|---|---|---|---|
| 主题进入时间 | 从开始表达至明确主旨出现的时间 | 秒 | 分享、答辩、Pitch |
| 结构完整度 | 是否具备开头、主体、结尾 | 0–100 分 | 全场景 |
| 重点句占比 | 明确表达观点/结论/重点的句子占比 | % | 分享、答辩、Pitch |
| 结尾收束率 | 是否在结尾进行总结或回扣 | 0/1 或 0–100 分 | 全场景 |
| 结论前置率 | 关键结论是否在前段清晰呈现 | % | 答辩、Pitch |
| 词汇丰富度 | 去重词汇占全部词汇的比例 | TTR / 修正 TTR | 分享、Pitch |
| 语法错误密度 | 单位词数中的语法错误数量 | 每百词错误数 | 分享、答辩、Pitch |
| 表达简洁度 | 有效信息与冗余信息的比例 | 0–100 分 | 答辩、Pitch |
| 口头禅密度 | 口头填充词频率 | 次/分钟 | 全场景 |
| 专业术语准确率 | 专业术语使用是否正确恰当 | % 或 0–100 分 | 答辩、分享 |
| 语速 | 单位时间输出词数或字数 | 词/分钟 或 字/分钟 | 全场景 |
| 长停顿次数 | 超过阈值的明显停顿次数 | 次/分钟 | 全场景 |
| 自我修正率 | 回改、重启表达的频率 | 次/分钟 或 % | 即兴、面试 |
| 连续表达时长 | 不出现明显断裂的持续表达时间 | 秒 | 即兴、面试、主持 |
| 起步延迟时间 | 从收到任务到开始有效表达的时间 | 秒 | 即兴、面试 |
| 音量稳定性 | 音量是否持续处于合理区间 | 标准差 / 变异系数 / 0–100 分 | 全场景 |
| 发音清晰度 | 语音是否易于被听懂 | 0–100 分 | 全场景 |
| 语调变化度 | 音高与语调是否具备起伏 | Pitch variance / 0–100 分 | 主持、分享、Pitch |
| 注视镜头比例 | 面向镜头/观众方向的注视时长占比 | % | 全场景 |
| 低头看稿比例 | 低头或偏离听众视线的时长占比 | % | 分享、答辩、面试 |
| 姿态稳定性 | 身体姿态是否平稳 | 0–100 分 | 面试、答辩、主持 |
| 互动触发率 | 主动与听众建立呼应、提问、呼唤的频率 | 次/分钟 | 主持、分享 |
| 记忆点密度 | 金句、总结句、强化句出现的频率 | % 或 次/分钟 | 分享、Pitch |
| 回应针对性 | 是否直接回应问题或场景需求 | 0–100 分 | 答辩、面试 |

---

## 13. 各场景优先量化指标

| 场景 | 优先量化指标 |
|---|---|
| 主持 | 互动触发率、注视镜头比例、节奏波动指数、语调变化度、音量稳定性 |
| 主题分享 | 主题进入时间、结构完整度、重点句占比、表达简洁度、口头禅密度 |
| 即兴表达 | 起步延迟时间、长停顿次数、自我修正率、连续表达时长、结尾收束率 |
| 汇报答辩 | 结论前置率、重点句占比、回应针对性、专业术语准确率、表达简洁度 |
| 面试 / Pitch | 第一亮点出现时间、记忆点密度、注视镜头比例、语调变化度、流畅度 |

注：“第一亮点出现时间”可视为 Pitch 场景下的扩展结构指标。

## 14. 使用建议

### 14.1 用于评分
- V1.0 建议优先选择 **12–20 个核心评分指标** 构成评分内核。
- 不建议一次性将全部量化指标直接纳入总分。
- 核心指标负责稳定评分，辅助指标负责解释分数和生成建议。

### 14.2 用于反馈
- 实时反馈只使用少量高价值指标。
- 赛后报告展示一级维度结果与关键指标表现。
- 高级报告或历史页展示长期趋势、稳定性和短板修复情况。

### 14.3 用于产品呈现
建议前端优先展示以下三层：
1. **场景总分 + 六维雷达图**
2. **场景重点反馈**
3. **核心指标明细与历史趋势**

## 15. 结语

Speak Up 的核心不是判断“谁更会说”，而是识别：

**在某一个具体场景中，用户是否完成了该场景真正要求的表达任务。**

本文件通过“场景化评价体系 + 量化指标库”的整合，为 Speak Up 提供了从专业评价到产品化落地的统一基础。
